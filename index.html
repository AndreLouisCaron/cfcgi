Welcome to the cfcgi project home page!

Project description

cfcgi is a modern replacement for the reference FastCGI libraries. It features a streaming/incremental parser for FastCGI records written in plain C, as well as a higher-level C++ API. It provides the necessary infrastructure for implementing FastCGI support in both HTTP gateways (web servers) and application servers.

Features

  1. Tiny footprint! Statically compile cfcgi into your gateway or application with minimal overhead. At its minimum, cfcgi consists only of a very small C API.
  2. Streaming/incremental parser in C. The parser is fully interruptible, and thus suitable for use with asynchronous I/O. Just feed the parser data as it is received and let it tell you when it has data to process.
  3. High-level FastCGI application handlers in C++. Don't want to fuss with all that "complicated" C interface? Fine, just inherit from fcgi::Responder, fcgi::Authorizer or fcgi::Filter and create a FastCGI application handler in minutes.
  4. Full support for all FastCGI features, including authorizers and filters. Add custom authorization without the fuss of writing a plug-in for your web server.
  5. Built-in FastCGI application drivers for popular environments. Mix and match your application handlers with different I/O models and benchmark them. See which one gives you the best performance for your specific application.

Getting started

Documentation

The Doxygen API documentation will be pre-compiled and made available online shortly.

Checking out the code

Pop your favorite terminal, locate your favorite development folder (named projects here) and fetch the code and its dependencies:

$ cd projects
$ git clone git@github.com:AndreLouisCaron/cfcgi.git
$ cd cfcgi
$ git submodule init
$ git submodule update

Compiling

UNIX-like systems

$ mkdir work && cd work
$ cmake ..
$ make
$ make test

Microsoft Windows

If you're used to the command line interface and a real text editor, you can build the software using NMake:

$ mkdir work && cd work
$ cmake -G "NMake Makefiles" ..
$ nmake
$ nmake /A test

Alternatively, if you prefer the Visual Studio IDE and/or want to use the interactive debugger, you can generate Visual Studio solutions instead:

$ cmake -G "Visual Studio 9 2008" ..
$ devenv

Use "Visual Studio 8 2005" or "Visual Studio 8 2005 Win64" for 32-bit and 64-bit builds using Visual Studio 2005.
